# project_values.md

Our project values

## Guidelines

- Type-safety is a core value of our project.
- We prefer performance overall.
- Use features already implemented in the project, do not repeat work on every module.
- Prefer composition over inheritance.
- Prefer immutability over mutability.
- We're big fans of functional programming. We prefer pure functions over impure ones.
- We'll do have use of the `returns` library for our project. The documentation is available [returns](../../.reference/docs/returns/docs/index.rst).
- We're an `async` first project. We'll do heavy use of `asyncio`, `asyncstdlib` and other `async` libraries.
- We're using `uv` as our package manager.
